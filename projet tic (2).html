    L1) Les techniques de l’information et de la communication (TIC) Les TIC regroupent les outils matériels et logiciels qui permettent de communiquer, accéder aux informations, stocker, manipuler et produire des textes, des images, des sons et des vidéos. Les TIC résultent de la convergence de l'informatique, des télécommunications, des électroniques et de l'audiovisuel et utilisent les techniques de l'informatique et d'internet, de l'audiovisuel, du multimédia et des télécommunications.

    2) Catégories et objectifs Les TIC sont classées en 3 catégories: téléinformatique, télécommunications et électronique grand public. Les TIC ont pour but de permettre la connectivité, la modularité, une implantation simple, une utilisation facile et la fiabilité.

    3) Composants de TIC La partie matérielle des TIC comprend des appareils tels que les PC portables, les télévisions, les téléphones portables, les tablettes interactives, les appareils photo numériques et les robots. Les stations de travail sont des machines capables d'envoyer des données vers les réseaux. La partie logicielle inclut les systèmes d'exploitation, les logiciels d'application et les réseaux tels qu'Internet ou mobiles.

    4) Caractéristiques : L'utilisation des TIC permettrait de réduire le temps perdu pour les déplacements, de décentraliser le pouvoir, de faciliter l'accès aux informations pour l'ensemble du personnel, de rationaliser la communication pour une plus grande performance de l'organisation, de participer à l'activité de la mondialisation économique, de permettre une répartition de l'information entre un grand nombre de stations avec une qualité excellente et de transmettre des informations à tout moment et de n'importe où.

    5) Initiation aux techniques de communication Les objectifs de communication se définissent selon trois dimensions : cognitif, affectif et comportemental. Le but est de provoquer la prise de conscience, l'attention, la notoriété et l'information au niveau cognitif, de modifier ou renforcer l'image de produit au niveau affectif et de modifier les comportements en provoquant l'achat ou l'essai au niveau comportemental. Les partenaires de communication :sont l'émetteur qui transforme le message en signaux et le récepteur qui reçoit les signaux et reconstruit le message. Les vecteurs: sont le message émis par l'émetteur qu'il va tenter de coder le plus parfaitement possible et les médias qui sont le moyen utilisé pour transmettre le message. Les fonctions :sont le codage traduit le message en signifiants. Le décodage interprète le message. La réponse est l'avis du client et le feedback est la réaction du récepteur. Le bruit peut perturber la transmission. Bruit : il peut venir perturber les signaux pendant la transmission. Les types de communication :Il y a deux types de communication: la communication institutionnelle (corporate) qui consiste à communiquer sur l'image de marque de l'entreprise ou de l'institution, et la communication commerciale ou de produit qui met en avant les caractéristiques, les performances et les avantages du produit dans une communication crédible et cohérente. Les schémas de communication :Il y a trois types de communication: la communication interpersonnelle pour parler à un collaborateur, la communication de groupe pour parler à plusieurs personnes, et la communication de masse pour la publicité. Média et Multimédia :Les médias sont des moyens impersonnels de diffusion collective d'informations, tels que la presse, la radio, la télévision, le cinéma et Internet.     Les composants matériel d’un réseau informatique :

    1-Les cartes réseau : également appelées cartes d'interface réseau (NIC), permettent aux ordinateurs de se connecter à des réseaux tels qu'Internet ou les réseaux locaux. Les cartes réseau sont installées à l'intérieur de l'ordinateur et sont généralement connectées à une prise Ethernet ou Wi-Fi. Les cartes réseau sont essentielles pour la communication entre ordinateurs sur un réseau.

    2-Switch: Un switch (ou commutateur réseau) est un composant matériel qui permet à plusieurs appareils tels que des ordinateurs, des imprimantes, des serveurs, etc. de communiquer entre eux en utilisant des adresses MAC (Media Access Control). Le switch stocke les adresses MAC dans une table de correspondance et envoie les données uniquement à l'appareil destinataire, plutôt que de les diffuser à tous les appareils du réseau. Les switches sont disponibles en différentes tailles, allant des petits switches pour les réseaux domestiques et les petits bureaux, aux switches de niveau entreprise pour les réseaux plus importants.

    3-Hub: Un hub (ou concentrateur) est un composant matériel utilisé pour connecter plusieurs appareils sur un réseau local (LAN). Le hub fonctionne comme un répéteur, il reçoit les données envoyées par un appareil et les transmet à tous les autres appareils connectés. Contrairement à un switch qui envoie les données uniquement à l'appareil destinataire, le hub envoie les données à tous les appareils connectés. Les hubs ont été largement remplacés par des switches, qui sont plus efficaces pour la gestion du trafic réseau et permettent une communication plus rapide et plus sécurisée entre les appareils connectés.

    4/ Le modèle OSI : Osi: OSI (Open Systems Interconnection) est un modèle de référence pour les communications de réseau qui divise les communications en sept couches distinctes. Chaque couche est responsable de fonctions spécifiques liées à la communication de données entre les appareils de réseau. Les sept couches du modèle OSI sont la couche physique, la couche liaison de données, la couche réseau, la couche transport, la couche session, la couche présentation et la couche application. Le modèle TCP : Le modèle TCP/IP est un modèle de référence pour les communications de réseau. Contrairement au modèle OSI, qui divise les communications en sept couches distinctes, le modèle TCP/IP divise les communications en quatre couches principales : la couche liaison de données, la couche réseau, la couche transport et la couche application.

    5/L’adressage : Une adresse IP (Internet Protocol) est un numéro qui identifie chaque ordinateur dans les réseaux qu'il appartient. Il y a deux modes d'adressage : IPV4 et IPV6. L'adresse IPV4 est composée de 4 octets, soit 32 bits. Chaque paquet contient une adresse source et une adresse de destination, chacune sur 32 bits. Les adresses IPV4 sont exprimées en décimale en utilisant 4 chiffres séparés par des points.L'adresse IPV6 est un numéro codé sur 128 bits, représenté en 8 mots de 16 bits séparés par des «:», chacun en hexadécimal. Chaque adresse IPV4 est composée de deux parties : une partie pour l'adresse réseau et une partie pour l'adresse hôte. La partie réseau est constituée des bits de valeur supérieure, tandis que la partie hôte est constituée des bits restants. On peut reconnaître qu'un réseau est un {hôte} lorsque la valeur des bits pour la partie réseau est la même. Le nombre de bits contenus dans la partie hôte détermine le nombre de machines possibles sur ce réseau. Les 32 bits sont divisés en 4 octets, et chaque octet est converti en décimal. Il y a deux types d'adresses IPV4 : l'adresse réseau et l'adresse de diffusion. L'adresse réseau est une adresse dans laquelle tous les bits de la partie hôte sont égaux à 0. Tous les hôtes (machines) de ce réseau ont la même partie réseau. L'adresse de diffusion est une adresse dans laquelle tous les bits de la partie hôte sont égaux à 1. Cette adresse permet à un PC d'envoyer des paquets à l'adresse de diffusion de réseau, et tous les hôtes de cette adresse reçoivent ces données, sans qu'il distribue un paquet pour chaque PC d'une manière unique. L’adresse hôtes : On attribue les valeurs situés entre l’adresse réseau et l’adresse de diffusion sur {machines} situé sur cette réseau.

    resumer chapitre 5

    Git est un logiciel de contrôle de version essentiel pour la gestion de projet et la collaboration. Il permet de conserver l'historique des modifications et de revenir à des versions antérieures si nécessaire.

    Un système de contrôle de version est utilisé pour suivre les changements d'un projet, éviter les conflits lorsqu'il y a plusieurs contributeurs et faciliter la coordination des modifications.

    Il existe deux modèles de systèmes de contrôle de version : centralisé et décentralisé. Dans le modèle centralisé, le code est stocké sur un serveur central, tandis que dans le modèle décentralisé, chaque utilisateur dispose d'une copie complète du code.

    GitHub est un service en ligne qui héberge des dépôts Git, permettant aux développeurs de stocker, collaborer et contribuer à des projets.

    Git peut être utilisé en ligne de commande ou avec une interface graphique. Ce cours se concentre sur l'utilisation en ligne de commande pour accéder à toutes les commandes Git et assurer une cohérence entre les plateformes.

    Pour installer Git, téléchargez la dernière version depuis le site officiel et suivez les instructions.

    Après l'installation, configurez Git en définissant votre nom d'utilisateur et votre adresse e-mail à l'aide de la commande "git config".

    Un dépôt Git est une copie des fichiers d'un projet et de leur historique des versions. Il peut être créé en important un répertoire existant ou en clonant un dépôt Git existant.

    Git enregistre les données sous forme d'instantanés ou de snapshots. Chaque instantané représente l'état d'un projet à un moment donné.

    Les fichiers dans Git peuvent être suivis (dans l'instantané le plus récent) ou non suivis. Les fichiers modifiés sont des fichiers suivis qui ont été changés mais pas encore préparés ou validés.

    Git a trois principales zones : l'arborescence de travail (où les fichiers sont modifiés), la zone de préparation (où les modifications sont préparées pour le prochain commit) et le dépôt Git (qui stocke les méta-informations et la base de données du projet).
